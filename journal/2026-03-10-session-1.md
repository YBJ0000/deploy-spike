- **目标**：尝试「第二步：登录并创建项目」——在 Dokploy 里点击 Create project。

### 2026-03-10 — Compose 部署时 Select repository 找不到 medical-server

- **目标**：在 medical-server-stack 的 Deploy Settings 里选 GitHub 作为 Provider，并选择 medical-server 仓库以提供 compose 文件。
- **操作**：已用个人 GitHub 账户连接 Dokploy（Dokploy-2026-03-11-3sbry6）；在 Provider 选 GitHub 后，Github Account 已选该连接，但 **Repository 下拉框里没有 medical-server**。
- **结果**：无法选到 medical-server 仓库。
- **原因**：medical-server 属于**公司 GitHub 组织（Organization）**下的**私有仓库**，当前连接的是**个人 GitHub 账户**，OAuth/App 只授权了个人账号下的仓库，**看不到组织里的私有 repo**。
- **结论**：**Confirmed**。Dokploy 的「Select repository」只列出当前已连接账户有权限的仓库；组织私有库需由组织安装/授权同一 GitHub App，或使用能访问该组织的账号/方式。
- **下一步**：采用下面任一方案取得 compose 来源——用个人有权限的仓库（如 deploy-spike）、或 Raw 粘贴、或组织授权 / Fork（见 operating-guide 与 findings 文档）。

### 2026-03-10 — 疑问：只设置了 Compose Path，点 Deploy 如何拉取镜像/代码？

- **疑问**：deploy-spike 里只有文档和 yml，没有存镜像也没有 medical-server 代码；在 Dokploy 里选 deploy-spike + Compose Path 后点 Deploy，Dokploy 如何拉取镜像或代码？是否还要在本地构建 medical-server 的镜像并推到 Docker Hub？
- **解答**：  
  - Compose 里**没有** `build:`，只有 **`image:`**（mongo:8、rabbitmq:3-management、redis:7-alpine、ghoshorn/raidar:server-latest）。  
  - 点 Deploy 时，Dokploy 会：(1) 从 GitHub 拉取 **compose 文件本身**（deploy-spike 的 `configs/docker-compose-medical-server.yml`）；(2) 在服务器上按 YAML 执行 **`docker pull`** 上述镜像并启动容器。  
  - 因此**不需要**在 deploy-spike 或 medical-server 里存镜像或代码，也**不需要**你本地构建并推镜像；Raidar 已以 `ghoshorn/raidar:server-latest` 存在于 Docker Hub。  
  - **额外注意**：若 `ghoshorn/raidar` 为 Docker Hub **私有仓库**，需在 Dokploy 的 Settings → Registries 中配置 Docker Hub 账号，否则 pull 会失败。
- **结论**：**Confirmed**。保存 Provider/Repository/Compose Path 后直接点 Deploy 即可；若 raidar 服务拉取失败，检查私有镜像仓库登录配置。

### 2026-03-10 — 方案 3 步骤 2：bootBuildImage 在 EXPORTING 报错 content digest not found

### 2026-03-10 — 清理缓存后出现平台不匹配（linux/amd64 vs linux/arm64）

- **现象**：按文档执行 `docker image prune -a -f` 后预拉 `run-jammy-base`、`builder-jammy-base`，再执行 `./gradlew bootBuildImage ...`，报错：`Image platform mismatch detected. The configured platform 'linux/amd64' is not supported by the image '...run-jammy-base:latest'. Requested platform 'linux/amd64' but got 'linux/arm64'`。
- **原因**：在 Mac（Apple Silicon）上未带 `--platform` 的 `docker pull` 会拉取 **arm64** 镜像；构建侧请求的是 **linux/amd64**（便于在常见 VPS/Dokploy 上运行），故平台不一致。
- **处理**：(1) 在 `medical-server/app/build.gradle` 的 `bootBuildImage` 中显式设置 `imagePlatform = "linux/amd64"`；(2) 在 [docs/build-and-push-raidar-image.md](../docs/build-and-push-raidar-image.md) 中新增「步骤 2 平台不匹配」小节，要求预拉时使用 `docker pull --platform linux/amd64 ...`，清理后重试也须带 `--platform linux/amd64`；(3) 常见错误表增加该条。
