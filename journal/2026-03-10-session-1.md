- **结论**：**Confirmed**。官方安装路径仅支持 Linux VPS；前置条件「已有一台可 SSH 的 Linux 服务器」为必须。

### 2026-03-10 — 在 Dokploy 界面创建项目失败 + 自托管 vs Cloud 结论

- **目标**：尝试「第二步：登录并创建项目」——在 Dokploy 里点击 Create project。
- **操作**：在浏览器中打开 Dokploy 并点击创建项目，出现错误：「Error creating the project: No servers available, Please subscribe to a plan」；随后查看 Billing 页面，看到 Hobby / Startup / Enterprise 等付费计划。
- **结果**：创建项目失败。当前使用的是 **Dokploy Cloud（托管版）**，需订阅才有可用 Server；这与 issue #157 要求的「self-hosted, not a paid hosted solution」**不符合**。
- **证据**：截图（Create project 报错 + Billing 定价页）。
- **结论**：**Confirmed**。不应通过订阅 Startup/Hobby 继续；应使用**自托管版**（在自己的 Linux 上安装，不向 Dokploy 付费）。自托管版安装完成后，在同一台 Linux 上会有可用 Server，创建项目不会提示「subscribe to a plan」。Dokploy 自托管版提供 Web 终端（Settings → Servers → Enter Terminal），不必一定用 SSH；但安装脚本仍须在 Linux 上执行（可用 Mac 上的 Multipass/UTM 起 Linux VM 获得等效环境）。
- **下一步**：不订阅 Dokploy Cloud。选择「租 Linux VPS」或「Mac 上 Multipass/UTM 起 Linux VM」，在 Linux 内执行安装脚本，再访问自托管实例的 `http://<Linux IP>:3000` 完成第二步。

### 2026-03-11 — Dokploy 上 MongoDB rs0 初始化踩坑记录

- **目标**：在 Dokploy 的 medical-server-stack 上完成 MongoDB 副本集 rs0 的初始化。
- **操作**：
  - 按最初文档尝试在 Dokploy 界面点击 Compose 栈上的「Open Terminal」，希望进入 `mongodb` 容器并执行 `mongosh` / `rs.initiate()`。
  - 在该终端中执行 `ps aux | grep mongod`，发现只有 `grep` 自己，没有 `mongod` 进程；执行 `mongosh` 报 `command not found`。
  - 回看 `configs/docker-compose-medical-server.yml`，确认 Mongo 通过 `ports: "27017:27017"` 暴露到 Dokploy VM；改为在本机终端直接执行 `mongosh "mongodb://192.168.64.4:27017"` 成功连接到 MongoDB 8.2.5。
  - 随后在 `mongosh` 会话中执行 `rs.initiate()`，得到 `ok: 1` 且提示使用默认配置；再执行 `rs.status().ok` 返回 `1`，提示符变为 `rs0 [direct: primary]`。
- **结果**：成功。MongoDB 副本集 rs0 已在 Dokploy VM 上完成初始化，当前节点为 primary，可接受应用写入。
- **证据**：`mongosh "mongodb://192.168.64.4:27017"` 输出显示 Connected to MongoDB 8.2.5；`rs.initiate()` 返回 `ok: 1`；`rs.status().ok` 返回 `1`，shell 提示符显示 `rs0 [direct: primary]`。
- **结论**：**Confirmed**。Dokploy 的「Open Terminal」并不一定附着在具体服务容器（如 `mongodb`）上，更可靠的方式是在宿主机/开发机上使用 `mongosh` 通过 Dokploy VM 的私网 IP + 暴露端口（27017）连接 Mongo，再在该会话中完成 rs0 初始化和状态检查。
- **下一步**：回到 Dokploy 中对 `raidar` 容器执行 Restart，观察日志是否不再出现 `REPLICA_SET_GHOST` 或 `MongoTimeoutException`，并访问 `http://<VM私网IP>:8080/swagger-ui/index.html` 验证 Raidar API 是否正常启动。
